VPKit SDK Reference

Basic usage

VPKit provides the public interface for the framework. This is where you will find methods to initialise the SDK.

You will also need to initialise a VPKPreview (UIImageView sublcass) with a VPKImage (UIImage sublcass). VPKPreview has two delegates that may be set to adjust behaviour.

VPKVeepViewer and VPKVeepEditor are the view controllers for viewing and creating veeps. There is no need to invoke these objects explicitly - the VPKPreview can handle simple use cases internally.

Event notifications

A number of event messages are available to intercept as NSNotification objects.

See VPKEvent constants for a list of event names.

The notification objects include a userInfo dictionary with a single VPKIdentifierKey entry. If a mediaIdentifier string is set on the VPKPreview’s VPKImage, this will be returned as the value. The identifier may aid in ensuring that the correct media events are handled.

See VPKImage.mediaIdentifier
